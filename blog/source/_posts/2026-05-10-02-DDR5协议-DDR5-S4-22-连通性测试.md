## 4.22.0 不需要启动 DDR 就能检测虚焊

DDR5 DIMM 上有上百个焊点——CA[13:0]、DQ[7:0]、DQS、CK、CS_n、ODT、ALERT_n……任何一个出现开路或虚焊，系统都无法正常工作。但传统的故障定位方法要求先完成完整的 DDR 初始化（配置 MR、做训练），然后跑内存读写测试——这可能要几毫秒，而且如果 CA 总线上的某个引脚开路，初始化本身就可能失败。

CT Mode 用一个巧妙的方法绕过了所有复杂性：拉高 **TEN（Test Enable，专用测试引脚，正常操作时拉低）**——DRAM **完全旁路**正常的存储功能。所有引脚被重新分配为 Test Input / Test Output / Chip Select。内部逻辑变成一个**纯异步的组合 XOR 网络**——没有时钟、没有状态机、没有训练。

---

## 4.22.1 5 个 Min Term 的 XOR 网络

CT Mode 的核心是 5 个 Min Term（MT0~MT4），每个是多个引脚的 XOR：

- MT0 = XOR(CA[0,1,2,3,8,9,10,11]) — 8 个 CA 的 XOR
- MT1 = XOR(CA[0,4,5,6,8,12,13], ALERT_n) — 8 输入
- MT2 = XOR(CA[1,4,9,12], CK_t, CK_c) — 6 输入
- MT3 = XOR(CA[2,5,7,10,13], CK_t) — 6 输入
- MT4 = XOR(CA[3,6,7,11], CK_c, ALERT_n) — 6 输入

为什么用 XOR？XOR 的真值表对每个输入的变化都敏感——任何一根 Test Input 的电平翻转都会导致对应 MT 的输出翻转。如果某根 CA 线在 PCB 上开路（固定在某电平），那在所有 Test Pattern 下，对应 MT 的输出中与这根线相关的部分都不会变——故障就被定位了。

MT 值及其反相（MT_B）被映射到 DQ/DQS/DM 输出上。Controller 在 Test Input 上施加 Walking-1 Pattern，从 Test Output 上读取 MT 值，与预期值比较——不一致 = 找到了故障引脚。测试完成后必须 TEN 拉低 + RESET_n 复位 DRAM。
