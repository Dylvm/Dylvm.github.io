## 3.1 简化状态图：一张图理解 DDR5 的所有状态

Figure 3（JESD79-5D Page 9）是 DDR5 的"技能图谱"——它用一张图展示了 DRAM 可能处于的所有状态、以及用什么命令可以在状态之间跳转。理解这张图不需要死记硬背——只需要抓住几个核心状态簇：

**核心操作簇**（中间偏左）：Idle ⇄ Bank Active ⇄ Reading/Writing ⇄ Precharging。这是 DDR5 最常见的状态循环——CMD: ACT → Bank Active → RD/WR → 数据 → PRE → 回 Idle。

**训练簇**（下方偏右）：CS Training Mode (CSTM) → CA Training Mode (CATM) → Write Leveling → Read Preamble Training → VrefCA/CS Training。这是初始化阶段的标准训练序列。进入和退出都通过 MPC 命令（MPC+CSTM enter / MPC+CATM enter 等）。

**电源管理簇**（左下方）：Idle → PDE → Active Power Down / Precharged Power Down → PDX 退出；或者 Idle → SRE/SREF → Self Refresh / MPSM → SRX 退出。注意 Self Refresh 退出时 DRAM 需要 NOP+NOP+NOP 来恢复内部时钟。

**ZQ 校准簇**（右上角）：Idle ⇄ ZQ Calibration（通过 MPC+ZQ start 进入）。校准过程中 DQ/DQS 总线可用，但 CA 总线必须保持 DES。

**刷新簇**（右下方）：Idle → REFab/REFsb → Refresh → 完成 → 回 Idle。还包括 RFMab/RFMsb（刷新管理）——由 ALERT_n 或内部计数器触发。

## 3.2 基本功能：DDR5 的操作骨架

### 3.2.1 16n Prefetch 架构

DDR5 的 16n Prefetch 是理解所有时序的基础。内部 DRAM 核心以 300 MHz 工作，每个 CK 周期从阵列中取出 **16n bit 的数据**（x8 器件 = 128 bits）。这 128 bits 通过 8:1 SERDES 在 8 个 CK 周期（16 个半周期）内串行输出到 DQ——对应于 **Burst Length = 16**。一次读或写操作 = 内部一次 128-bit 传输 + 接口上八百半周期的串行输出。

协议原文："A single read or write operation consists of a single 16n-bit wide, eight clock data transfer at the internal DRAM core and sixteen corresponding n-bit wide, one-half clock cycle data transfers at the I/O pins."

### 3.2.2 Bank 组织

8Gb 起步密度：x4/x8 为 8 BG × 2 Bank/BG = 16 Banks。≥16Gb 密度：8 BG × 4 Bank/BG = 32 Banks。x16 始终是 x4/x8 的一半 BG 数量（4 BG × 2 或 4 BG × 4）。

### 3.2.3 操作流程

每次读写都是两段式：**ACTIVATE 命令**打开一行（提供 BG + BA + Row 地址）→ 等 tRCD → **READ/WRITE 命令**选择起始列（提供 Col 地址 + BL\* 选择 + AP 选择）→ 等 RL/CWL → 数据传输 → PRE 或 Auto-Precharge。在正常操作前，DRAM 必须按预定义的流程上电并初始化。

## 3.3 复位与初始化流程

### 3.3.1 MR 上电默认值（Table 9）

DDR5 在上电后、初始化前的默认状态由 Table 9 定义。这些默认值确保 DRAM 在最低频率下安全可用——等训练和配置完成后才切换到高性能模式：

| MR | 默认值 | 含义 |
|-----|--------|------|
| MR0 OP[1:0] | 00B | BL16（默认 Burst Length） |
| MR0 OP[6:2] | 00010B | RL(CL) = 26 @ DDR5-3200（保守的低速 CL） |
| CWL | WL = RL-2 | CWL = CL-2（所有 Speed Bin 通用规则） |
| MR6 OP[3:0] | 0000B | WR = 30ns 或 48nCK @3200 |
| MR6 OP[7:4] | 0000B | tRTP = 7.5ns 或 12nCK @3200 |
| MR10 (VrefDQ) | 0010_1101B | VrefDQ = 75% VDDQ（初始保守值） |
| MR11 (VrefCA) | 0010_1101B | VrefCA = 75% VDDQ |
| MR12 (VrefCS) | 0010_1101B | VrefCS = 75% VDDQ |
| MR15 (ECS ETC) | 011B | 256 次错误阈值 |
| MR32 (CK ODT) | Strap 决定 | Group A = RTT_OFF, Group B = 40Ω |
| MR33 (CA ODT) | Strap 决定 | Group A = RTT_OFF, Group B = 80Ω |
| MR34 (RTT_PARK) | 000B | RTT OFF |
| MR35 (RTT_NOM) | 011B = 80Ω | 读写标称终端 = 80Ω |

### 3.3.2 上电初始化序列（Section 3.3.1 + Figure 5）

DDR5 的上电初始化序列是一个有严格时序约束的步骤链。在训练之前，DDR5 必须经过上电、复位、MRS 配置、ZQ Calibration 四个阶段。下面沿时间线（Ta→Tb→...→Tl）完整走读：

**Ta — 开始上电**：任何电源首次达到 300 mV 的时刻定义为 Ta。在此之后，RESET_n 建议保持低电平（≤ 0.2 × VDDQ），所有其他输入可以为未定义。器件输出在此期间保持禁用。VPP 必须**不晚于 VDD** 达到标称值——VPP 应同时或更早上电。所有电源的上电斜坡时间（Tb − Ta）不能超过 tINIT0。

**Tb — 电源稳定**：VDD, VDDQ, VPP, VREF 全部达到各自规格范围。从 Tb 开始，RESET_n 必须维持低电平至少 **tINIT1**。

**Tc — 释放 RESET_n**：tINIT1 满足后，RESET_n 可以被释放（拉高）。在 RESET_n 释放前至少 **tINIT2** 时间，CS_n 必须先被拉低。

**Td → Te — CA 总线保持 NOP**：RESET_n 释放后，CA 总线必须保持高电平在 NOP 编码状态（tINIT3）。

**Te → Tf — 发 NOP/DES**：tINIT4 期间在 CA 总线上施加 NOP 命令。随后在 Te 到 Tk 之间，合法的命令（MRW、MPC、VrefCA、VrefCS）之间必须插入 DES。

**Tk — ZQ Calibration 完成**：ZQCal Start + ZQCal Latch 完成后，DRAM 已具备正确的 Ron 和 RTT 阻抗。此时可以进入 CS 和 CA 训练。

**Tl — CS 和 CA 训练完成**：tCS/CA Training 完成后，发 MRW 将 MR2 OP[4] 设为 1（使能 Multi-Cycle CS 的 Single-Cycle 模式），后续的 MPC/VrefCA/VrefCS 命令可以使用 1-Cycle 格式——大幅提升训练速度。

### 3.3.3 初始化中的重要约束

**CA 总线在 RESET_n 释放后的状态**（Note 2-3）：从 Td 到 Te 期间，CA 总线**必须保持在高电平的 NOP 编码**。从 Te 到 Tf 期间，CA 总线上必须施加 NOP。**在 CS/CA 训练完成之前，MRR 和 MRW 命令虽然合法但可能无法正确执行**（Note 4）——所以在 Tk 之后、Tl 之前发 MRW/MRR 时要加倍小心。

**Multi-Cycle CS Assertion 的作用**（Note 5-7）：在 ZQcal 和 CS/CA 训练完成前，MPC、VrefCA、VrefCS 命令必须使用 **Multi-Cycle** 格式（MR2 OP[4] = 0）——因为此时 CS_n 和 CK 的时序对齐还不确定，Multi-Cycle 保证了至少一个 CK 上升沿能正确采样。CS/CA 训练完成后，推荐 MRW 设 MR2 OP[4] = 1 切换到 Single-Cycle 模式，缩短训练周期。

### 3.3.4 带 DFE 训练的扩展初始化

Figure 7（JESD79-5D Page 19）展示了包含 DFE Training 的完整初始化流程——在标准序列的 CS/CA/WL/Read 训练之后，增加 DFE 使能和系数自适应。DFE 训练完成后需要验证 BER 并锁定系数。

## 3.4 寄存器定义：MRR 与 MRW 的读写规范

### 3.4.1 寄存器访问协议

DDR5 改变了 DDR3/DDR4 的寄存器访问方式——从传统的"地址位映射"改为 **LPDDR 风格的"MRA + OP"包格式**。MRW（Mode Register Write）命令在 CA 总线上同时携带 **Mode Register Address（MRA[7:0]）**和**操作数（OP[7:0]）**。

**MRW 命令**：第 1 周期 CA[11:4] = MRA[7:0]（选择 MR0~MR255 中的某一个），第 2 周期 CA[7:0] = OP[7:0]（写入 8-bit 数据）。CW 位（第 2 周期 CA11）控制条件执行——CW = L 执行写入，CW = H 忽略（实现 Per-DRAM 条件写入）。

**MRR 命令**：第 1 周期编码同 MRW（但 CA[5:4] = LH 以区分），DRAM 将目标 MR 的值通过 **DQ[7:0] 输出**（伴随 DQS），格式为 BL16（不随 MR0 Burst Length 改变）。

### 3.4.2 可读可写性与 RFU 处理

每个 MR 的位标记为 **R**（只读）、**W**（只写——读出时该位固定返回 0）、**R/W**（可读写）或 **SR/W**（Status Read / Write——DRAM 读出时返回内部状态而非 Host 写入值）。

当整个 MR# 标记为 RFU（Reserved for Future Use）时，该 MR 未定义——DRAM 不保证读写操作有效，且可能不支持该寄存器。当某个已定义 MR# 中的个别位为 RFU 时，Host **必须向这些位写 0**。

对于只用于 x16 配置的 MR#——在 x4/x8 器件中该 MR# 全部视为 RFU。对于部分位字段用于不同配置/密度的 MR——Host 可以写入，但 DRAM 不会受这些位的影响。

### 3.4.3 MRR/MRW 时序约束

MRW → MRW：最小间隔 = **tMRD**（Mode Register Delay）。MRW 后等 tMRD 才能发下一条需要 MR 生效的命令。
MRR → MRR：提供特殊的 Burst 格式（BL16，数据在 DQ[7:0] 输出）。
MRR/MRW 与 VrefCA/VrefCS 命令的时序在 ODT 禁用和使能两种情况下有独立的间隔规定（Tables 22-23）。
