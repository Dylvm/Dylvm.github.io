> 本文是 SDRAM 基础系列的第一篇。我们从最底层的 1T1C 单元出发，理解 DRAM 如何用一个晶体管和一个电容记住一个 bit，再将几千万个这样的单元组织成可高效寻址的矩形网格。

---

## 1.1 DRAM 与 SRAM 的分岔路口

计算机内存有两种根本不同的技术路线。它们的分岔点在于**用什么来"记住"一个 bit**。

**SRAM（Static RAM）** 用一个由六个晶体管组成的**双稳态触发器**来存储数据——一旦写入就会一直保持，只要不掉电。好处是速度快（1\~2 ns 量级），不需要维护。代价是六个晶体管占用的硅片面积很大。所以 SRAM 适合做 CPU 内部的 Cache（KB\~MB 级），但不适合做 GB 级主存。

**DRAM（Dynamic RAM）** 走了另一条路。它用一个**电容器**来存储数据——有电荷 = 1，无电荷 = 0。每个单元只需要 **1 个晶体管 + 1 个电容**（1T1C），比 SRAM 省了六倍的晶体管数。但代价是：

1. 电容器会**漏电**——电荷几十毫秒就会流失，所以需要**定期刷新**（Dynamic 的含义）
2. 电容存储的电荷量极小（飞法级）。读操作时电容与位线共享电荷——位线电压变化只有 **几十到一百毫伏**，远小于数字电路能直接识别的逻辑电平，需要**高度灵敏的放大器**（Sense Amplifier）

这两个事实——**漏电需要刷新**、**信号微弱需要放大**——是 DRAM 所有设计的出发点。

## 1.2 1T1C 单元结构

一个 DRAM 存储单元由三个部分组成：

<div align="center"><img src="/images/ddr/1t1c-cell-structure.png" width="300" alt="Basic 1T1C DRAM cell structure" /></div>

> **图 1.2**: Basic 1T1C DRAM cell structure

**Access Transistor（存取晶体管）**是一个 NMOS 管。它的 Gate 接到 **Wordline（字线，WL）**，Source 接到 **Bitline（位线，BL）**，Drain 接到 **Storage Node（存储节点）**。NMOS 管的特性很简单：Gate 电压 > Source 电压 + Vth（≈ 0.7V）→ 导通；反之 → 关断。所以 WL 就是"开关控制线"：高电平 → 电容和 BL 连通；低电平 → 电容被隔离，电荷困在里面。

**Storage Capacitor（存储电容）**的电容值随工艺微缩持续变小。DDR3/DDR4 时代典型值为 **20\~30 fF**（飞法，10⁻¹⁵ 法拉），进入 DDR5 的亚 10nm 节点后进一步缩小到约 **5\~10 fF**——手指靠近金属焊盘的寄生电容就有几 pF（10⁻¹²），比 DDR5 电容大三个数量级。DDR5 引入片上 ECC 的直接原因之一就是：电容太小 → 存储的电子太少 → 随机单 bit 翻转概率显著升高。

## 1.3 存储电容的物理实现：向第三维度要面积

1T1C 的原理很简单——一个晶体管加一个电容。但"电容"在芯片上到底长什么样？这是 DRAM 工艺最核心的挑战：**如何在极小的硅片面积上做出一个"够大"的电容？**

电容值 C = ε × (A/d)。在平面工艺中，A 受限于 2D 平面面积——要做到 20-30 fF 需要巨大的平面面积，不可接受。DRAM 的解决思路是**向第三维度要面积**——让电容"站起来"，在垂直方向获得更大的极板表面积。

<div align="center"><img src="/images/ddr/trench-capacitor.png" width="550" alt="Trench capacitor structure" /></div>

> **图 1.3**: Trench capacitor structure

**沟槽式电容（Trench Capacitor）**：在硅衬底中**向下刻蚀深沟槽**（深度可达数微米——相当于向下挖了"几十层楼深"），在沟槽内壁上沉积介电层和多晶硅电极。沟槽的侧壁面积远大于平面面积——相当于把平面电容"卷"成圆筒塞进硅片。1990\~2000 年代广泛使用，IBM、Infineon 是主要推动者。

<div align="center"><img src="/images/ddr/stacked-capacitor.png" width="550" alt="Stacked capacitor structure" /></div>

> **图 1.4**: Stacked capacitor structure

**堆叠式电容（Stacked Capacitor）**：在硅表面**上方**——在 Access Transistor 顶部——构建**三维"皇冠"或"柱状"结构**。电容"站"在晶体管上方，不占用额外硅衬底面积。这是现代 DRAM 的主流方案（三星、SK hynix、美光），与先进逻辑工艺更兼容。

**为什么这对理解 DRAM 很重要？** 因为电容值决定了 δV。每一代工艺微缩，平面面积减半，但电容值必须保持足够大——否则误码率太高。沟槽式靠挖得更深，堆叠式靠站得更高——两者都在"对抗平面微缩"。这也是 DRAM 工艺总是落后逻辑工艺 1-2 代的原因：做这种 3D 电容需要特殊工艺步骤。

---

## 1.4 写操作：把数据"灌"进电容

写操作最直观：

**写"1"**：Controller 先把 BL 驱动到 Vcc（DDR5 = \~1.1V），然后 WL 导通。BL（Vcc）通过 NMOS 管给 Cs 充电，直到 Cs 两端电压接近 Vcc。WL 关断，电荷被"锁"在电容里。

**写"0"**：Controller 把 BL 驱动到 0V，WL 导通。Cs 上如果有电荷，就通过 NMOS 管放电到 BL。WL 关断，电容保持放电状态。

关键细节：NMOS 管导通时并非理想 0Ω 开关——导通电阻 Ron 约几 kΩ。电容充放电需要时间。这就是 **tWR（Write Recovery Time）** 的物理根源：如果电容还没充满就关 WL，数据处于"半高不低"的模糊状态，后续读出会出错。

---

## 1.5 读操作：破坏性读取

读操作是 DRAM 最核心的挑战。与写操作不同——读是**破坏性**的：电容 Cs 与 BL 的寄生电容 CBL 发生电荷共享后，Cs 上的电荷被"稀释"。

用一个比喻：Cs 是**小水桶**（25 毫升），BL 是**大水缸**（250 毫升——一根 BL 接了几百到几千个 Cell，寄生电容累积是 Cs 的 10 倍）。阀门（Access Transistor）打开时，小桶和大缸的水位"中和"——桶里的水被缸稀释了。

根据电荷守恒：δV ≈ Cs/(Cs+CBL) × (Vs-VBL) ≈ 25/(25+250) × (1.1-0.55) ≈ **50 mV**——不到供电电压的 5%。

这引出两个需求：① 需要**高度灵敏的放大器**（Sense Amplifier）将 50mV 放大到 Vcc/GND（第 02 章）；② 放大后必须**将数据写回电容**（Restore）——这就是"读即刷新"的本质。

读操作的完整四阶段（Precharge→Access→Sense→Restore）以及 SA 电路详解将在第 03-04 章展开。这里只需记住：**每次读操作都破坏数据，读完之后必须回写——这是 DRAM 一切时序约束的物理根源。**

---

## 1.6 电容器会漏电——所以需要刷新

"Dynamic"的含义就在这里。存储在 Cs 上的电荷通过三种渠道流失：PN 结漏电流（主要）、亚阈值漏电、电容介质漏电。温度每升高约 10°C，漏电速度约翻倍。

JEDEC 规定：**所有 Cell 必须在 64ms（Tcase ≤ 85°C）内至少被刷新一次。** 在更高温度下（85\~95°C），保持时间缩到 32ms——刷新频率加倍。刷新的具体机制将在第 02 章展开。

---

## 1.7 为什么是 1T1C 而不是 3T1C？

DRAM 历史上还有另一种单元——**3T1C**（三个晶体管 + 一个电容）。它的读操作是**非破坏性**的（读晶体管和写晶体管分开，不需要 Restore），随机读比 1T1C 更快。Intel 1970 年第一款商用 DRAM（1103）就用的 3T1C。

但它被淘汰了。原因只有一个：**面积**。1T1C 只有 1 个晶体管，3T1C 有 3 个——同样硅片面积，1T1C 塞进 3 倍容量。在 DRAM 行业，**密度（每 bit 成本）是一切**。速度可以通过 Bank 交错、Prefetch 来弥补，单元面积决定了晶圆上能切出多少颗芯片。

## 2.1 从单元到矩阵：曼哈顿类比

现在有了 1T1C 单元。问题是：**几千万个单元怎么组织在一起？**

《Memory Systems》用了一个极好的类比。想象曼哈顿——大道（Avenue）南北走向，街（Street）东西走向。指定会合地点只需说"第 5 大道和第 42 街的交叉口"——位置唯一确定。

DRAM 阵列完全一样。每一**行（Row）**类比于一条**街**——同一行的所有 Cell 连到同一根导线，叫 **Wordline（字线）**。每一**列（Column）**类比于一条**大道**——同一列的所有 Cell 连到同一对导线，叫 **Bitline（位线）**。

**选定一根 Wordline + 选定一对 Bitline = 唯一确定一个存储单元。**

```
          Col 0    Col 1    Col 2    ...   Col 1023
          ┌───┐    ┌───┐    ┌───┐          ┌───┐
 BL[0]    │ ● │    │ ● │    │ ● │          │ ● │    ← 每个 ● 是一个 1T1C 单元
/BL[0]    │   │    │   │    │   │          │   │
          │   │    │   │    │   │          │   │
WL[0] ────┼───┼────┼───┼────┼───┼──────────┼───┼── Row 0 (选中)
          │   │    │   │    │   │          │   │
WL[1] ────┼───┼────┼───┼────┼───┼──────────┼───┼── Row 1
          │   │    │   │    │   │          │   │
          └───┘    └───┘    └───┘          └───┘
          SA[0]    SA[1]    SA[2]         SA[1023]
```

在这个矩阵中：
- **同一行的所有 Cell 共享同一根 Wordline**。WL 被激活时，整行 Cell 同时"开门"——各自电容连接到各自的 Bitline。**这就是 ACT 命令的粒度是"整行"的原因——一个 ACT 打开的不是一个 Cell，而是几千个 Cell 同时。**
- **同一列的所有 Cell 共享同一对 Bitline 和一个 SA**。同一列上不同 Cell 不能同时被访问——电容都挂在同一对 BL 上。这就是同一 Bank 同一时间只能有一个 Row 打开的原因。

---

## 2.2 为什么行数 >> 列数？

DRAM 阵列中行数通常远多于列数——典型阵列 4096 行 × 1024 列，DDR5 16Gb 下可达 131072 行。为什么不是方的？

原因在**Bitline 的寄生电容 CBL**。每根 BL 连接到这一列上**所有** Cell——每个 Cell 的 Drain 贡献一点寄生电容。**行数越多 → 一根 BL 上挂的 Cell 越多 → CBL 越大 → δV 越小。** δV 太小（< 30mV）SA 就无法可靠判别。

而**列数**每增加一列 → 增加一对 BL + 一个 SA → 面积增加。所以列的规模受限于面积预算。

物理权衡：**行数受限于可接受的 δV，列数受限于面积预算。** 现代 DRAM 的行数从几千（FPM）到几十万（DDR5）。

---

## 2.3 一场 ACT 的"连锁反应"

理解了一条 ACT 命令发出后 DRAM 内部发生了什么，就理解了为什么 SA 是整个芯片的核心：

1. **Row Decoder 收到行地址** → 译码 → 驱动一根 WL 为高电平（升压到 Vcc+Vt，确保 NMOS 无损失传递 Vcc）
2. **整行所有 Cell 的 Access Transistor 同时导通** → 每个 Cell 电容与对应 BL 电荷共享 → 每根 BL 电压 \~50mV 偏离
3. **整行所有列的 SA 同时开始工作** → 感知 → 放大 → **锁存整行数据**
4. 此时，整行数据（几千 bit）"缓存"在 SA 阵列中——SA 阵列 = **行缓冲器（Row Buffer）**

行缓冲器有两个管理策略，影响性能和功耗的权衡：
- **开页策略（Open-Page）**：SA 保持活跃，后续访问同一行不需再次 ACT。功耗高，适合高局部性负载。
- **关页策略（Close-Page）**：SA 在回写完成后立即预充电释放。功耗低，适合低局部性负载。

## 2.4 x4 / x8 / x16 是怎么实现的？

在 FPM DRAM 时代，答案是**并联多个独立 Memory Array**——x4 就是 4 个 Array，x8 是 8 个，各自负责一个 DQ bit。现代 DDR4/DDR5 已不同：依靠**更宽的内部数据通路 + Prefetch + SERDES**：

- **DDR5 x8**：内部一次 Prefetch 取出 **16n × 8b = 128bit** 数据，通过 **8:1 SERDES** 串行化到 8 根 DQ 上——每根输出 16 个数据拍（BL16）
- **x4**：内部数据通路更窄（4bit），需要 BL32（32 拍）才能达到同样数据量（128b）
- **x16**：两组独立的 8bit 数据通路（Lower + Upper Nibble），各有自己的 DQS——本质是"两个 x8 拼在一起"

逻辑上仍是 N 个并行数据通路支持 N 根 DQ，但物理实现已演变为一个更大的内部分块阵列 + 更宽数据总线 + SERDES。
