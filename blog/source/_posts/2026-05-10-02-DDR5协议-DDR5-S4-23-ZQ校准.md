## 4.23.0 240Ω 电阻为什么是 DDR5 系统中最精密的元件

DDR5 芯片的数据输出驱动器（Ron）和片上终端电阻（RTT）不是固定值——它们会随三个因素漂移。**工艺**：不同晶圆、不同批次的晶体管阈值电压和迁移率不同，同一设计的 Ron 可能在 35Ω 到 45Ω 之间走。**温度**：温度每变化 10°C，载流子迁移率变化约 4%——Ron 和 RTT 随之变化。**电压**：VDDQ 从 1.078V 掉到 1.045V（在 ±2.5% 容差内），驱动强度减弱，等效阻抗增大。

这三个因素叠加起来，Ron 可能在设计值的基础上漂移 ±15% 以上。如果 Controller 期望终端是 40Ω 但实际是 34Ω——信号反射系数从预期的接近零变成不可忽略——反射回来的信号叠加在当前 bit 上，眼图闭合。所以需要一个**不随 PVT 变化的"黄金参考"**来定期校准。

这个黄金参考就是连接在 ZQ 引脚和 VSS 之间的**240Ω ±1% 精密电阻**。这是整个 DDR5 系统中精度要求最高的无源元件。DRAM 内部的校准引擎将内部的上拉电阻（Pull-Up）与这个外部 240Ω 比较，调整 PU 阻抗直到匹配；然后用已校准的 PU 去校准下拉电阻（Pull-Down）——最后将校准好的 PU/PD 值复制到所有的 Ron 和 RTT 档位。

## 4.23.1 ZQCal Start 与 ZQCal Latch：完整校准 vs 快速刷新

ZQ 校准通过 MPC 命令发起，有两种模式：

**ZQCal Start** 启动一次**完整**的校准流程。DRAM 内部的校准引擎逐一比较 PU 与 ZQ 电阻、PD 与 PU，将结果应用到所有阻抗档位。完成一次 ZQCal Start 需要 **tZQCAL ≥ 1 μs**——这个时间跨度内 DRAM 不能发正常操作命令（CA 总线可以发 DES）。这是**最彻底的校准**，用于上电初始化、RTT 或 Ron 的 MR 配置变更后、或长时间运行后。

**ZQCal Latch** 是"锁存刷新"——它不重新做完整校准，只是将上次校准好的值重新加载到驱动器和终端电路中。用时仅为 **tZQLAT ≥ max(30ns, 8nCK)**——比 Start 快了 30 倍。Latch 在运行时周期性执行（比如每几百毫秒一次），用于补偿自上次校准以来的温度和电压漂移。

理解一个关键区别：如果你在 tZQCAL 还没完成时又发了一个 ZQCal Start，前一次的校准结果被丢弃——新的 Start 重启校准引擎。如果你在 tZQCAL 没完成时发 ZQCal Latch，锁存的可能是**前一次**已完成校准的值（可能是很久之前的）。如果你不发 Start 直接发 Latch——就是把上一次的值"重新锁存"一遍，快速补偿温漂。

## 4.23.2 多久校准一次？

校准间隔的计算公式如下：

```
ZQ 校准间隔 = 0.5 / (TSens × Tdriftrate + VSens × Vdriftrate)

TSens = max(dRTTdT, dRondT) ≈ 1.5%/°C
VSens = max(dRTTdV, dRondV) ≈ 0.15%/mV
```

代入一个典型场景：温度以 1°C/s 漂移，VDDQ 以 15 mV/s 漂移 → 校准间隔 ≈ 0.5 / (1.5% + 2.25%) ≈ 13.3 秒。如果环境更稳定（温度变化 0.1°C/s，电压变化 1 mV/s），间隔可以拉长到 100 秒以上。保守的做法是每 1~10 秒做一次 ZQCal Latch（快速），每 10~60 分钟做一次 ZQCal Start（完整）。

## 4.23.3 外部 ZQ 电阻的物理约束

- **阻值**：240Ω ±1%——精度直接决定校准精度。如果用 5% 的电阻，Ron 的校准结果就至少有 5% 的误差
- **连接**：ZQ 引脚 → 电阻 → VSS（地），走线尽量短——ZQ 引脚的寄生电容应 ≤ 5 pF，加上外部 PCB 走线和电阻的寄生，总容性负载 ≤ 30 pF
- **多 Rank 共享单电阻**：如果多个 Rank 的 ZQ 接在一起，控制器必须确保它们的 ZQ 校准时间**不重叠**——同时校准会导致两个 DRAM 同时驱动 ZQ 引脚，互相干扰
