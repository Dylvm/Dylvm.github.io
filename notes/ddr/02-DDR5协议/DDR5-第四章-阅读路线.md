# DDR5 第四章学习路线图

> JESD79-5D Section 4: DDR5 SDRAM Command Description and Operation
> 共 40 个小节，覆盖从基本的读写操作到复杂的训练和可靠性功能。

---

## 阅读建议总纲

第四章是从"协议规范"的角度解释 DDR5 的每一个操作——每条命令怎么发、有什么时序约束、DRAM 怎么响应。如果直接按 4.1 → 4.2 → ... → 4.40 的顺序读，会陷入大量琐碎的参数规定中而失去整体脉络。

一个更有效的阅读策略是**按功能主题分组**，先建立核心操作流的认知骨架，再逐步深入特殊功能和高级特性。

---

## 第一层：必须先读的基础（3 篇）

> 这是理解所有后续内容的前提。建议按顺序读完。

| 顺序 | 文档 | 为什么先读 |
|------|------|-----------|
| **1** | [DDR5-S4.1-命令真值表] | 这是 DDR5 命令系统的"字典"。你必须先知道 ACT 是什么意思、READ 的 CA 编码长什么样、1-Cycle 和 2-Cycle 命令怎么区分。后续每一节讲的命令，最终都要回到这张表来看编码。**建议读完前言+摘要后，不要求立刻背下所有编码，但至少要理解 H/L/V/X 符号约定、CA1 区分机制、以及表的基本结构。** |
| **2** | [DDR5-S4.6-激活命令] | ACT 是所有操作的起点——在读写任何数据之前，必须先用 ACT 打开一行。这一节非常短，但它是连接"银行状态管理"和"列操作"的桥梁。 |
| **3** | [DDR5-S4.3-预充电命令] | PRE 是 ACT 的"逆操作"——关闭一行，释放 Bank。理解 PREab/PREsb/PREpb 三种粒度 + Auto-Precharge 机制，是理解 Bank 状态管理的前提。 |

读完这 3 篇之后，你应该能回答：DDR5 的命令是怎么编码的？ACT 和 PRE 各自做什么？Bank 的状态是怎么变的？

## 第二层：核心读写操作（5 篇）

> 建立在第一层基础上，深入理解一次完整的读写访问。

| 顺序 | 文档 | 为什么此时读 |
|------|------|-------------|
| **4** | [DDR5-S4.2-突发长度] | 在真正读写之前，你得知道一次读/写操作到底传输多少数据。BL16、BC8、BL32 的区别是什么？Burst Order（数据拍顺序）怎么确定的？这一节为后续的读/写操作奠定了"数据量"的概念基础。 |
| **5** | [DDR5-S4.7-读操作] | 完整的读操作时序：ACT → tRCD → READ → tCL → DQS Preamble → 数据输出 → Postamble → tRTP → PRE。理解 Read Latency（RL）的构成、Auto-Precharge 的行为、以及同 BG/不同 BG 对命令间隔的影响。**这是第四章最重要的一节之一。** |
| **6** | [DDR5-S4.8-写操作] | 完整的写操作时序：ACT → tRCD → WRITE → tCWL → DQS Preamble → 数据输入 → Postamble → tWR → PRE。与读操作对比阅读——对称中有什么不对称？Write Latency（WL）为什么比 RL 短？Partial Write（WR_Partial）和 DM 怎么工作？ |
| **7** | [DDR5-S4.4-可编程前导后导] | 读和写都涉及 DQS 的 Preamble 和 Postamble。这一节解释了 MR8 如何配置这些过渡期的长度，以及不同 Speed Bin 下应该选什么配置。**阅读时机：读完 4.7 和 4.8 后，回来看 DQS 时序的具体参数。** |
| **8** | [DDR5-S4.5-间隔导言] | 连续两个 Burst 之间的 DQS 行为。无缝 Burst（Seamless Burst）、有间隙 Burst、以及读写切换时的 DQS turnaround。篇幅短，但理解连续访问场景下的 DQS 行为很重要。 |

读完这 5 篇之后，你应该能回答：一次完整的读/写操作从头到尾经过哪些步骤？Burst Length 如何影响数据量？DQS 在一开始和结束时是怎么过渡的？

| **10** | [DDR5-S4.9-自刷新操作] | 最深的低功耗状态——外部时钟可以停止，DRAM 自行刷新。理解进入/退出时序 + SRX 后的额外刷新要求 + Clock-Sync 功能。 |

> 正常运行时需要周期性执行的维护操作。

| 顺序 | 文档 | 为什么此时读 |
|------|------|-------------|
| **22** | [DDR5-S4.15-多功能命令] | MPC 是 DDR5 的"瑞士军刀"——ZQ 校准、训练模式进入、DQS 振荡器、1N/2N 切换等几十种功能都通过这一个命令扩展。理解 MPC 的 OP 码分类和工作机制，是把第四层和第五层串起来的纽带。 |
| **23** | [DDR5-S4.23-ZQ校准] | ZQ 校准输出驱动和终端阻抗——这是周期性维护的核心操作。理解 ZQ Start vs ZQ Latch、校准间隔的计算公式、240Ω 外部电阻的要求。 |
| **24** | [DDR5-S4.16-按DRAM寻址] | PDA（Per DRAM Addressability）——允许同一 Rank 内不同 DRAM 芯片有不同的配置。在 Multi-Rank 和 DIMM 场景中很重要。 |
