## 前言：为什么要从最底层开始？

学习 DDR 协议时，最常见的做法是直接看 JEDEC 标准中的时序参数表——tRCD、tRP、tRAS、tWR、tRFC——然后记住它们的数值和含义。这种做法可以快速上手，但缺少"物理直觉"：为什么 tRCD 是 ~14ns 而不是 5ns？为什么读操作要分四个阶段？为什么刷新周期是 64ms？

答案藏在 DRAM 芯片的内部电路中。DDR 协议中的每一个时序参数，追到底都是芯片内部某个模拟电路的工作时间——Sense Amplifier 放大微弱信号需要几纳秒，Bit Line 均衡需要几纳秒，电容充电需要几纳秒，加起来就是 tRCD、tRP、tWR……理解了这些物理过程，时序参数就不再是需要死记硬背的数字，而是可以从第一性原理推导出来的自然结果。

本文的目标是：**从一个电容开始，一步步搭建出一颗完整的 DRAM 芯片，让每一个时序参数和每一个协议规定都有清晰的物理直觉支撑。**

阅读本文不需要模拟 IC 设计背景。只需要知道晶体管可以当开关用、电容可以存电荷、以及欧姆定律——这些就够了。全文从第一性原理出发，每一步都回答"为什么"。

## 1. 从 SRAM 和 DRAM 的分岔路口开始

在半导体存储器领域，有两条截然不同的技术路线：SRAM 和 DRAM。它们的分岔点在于**用什么来存储 1 个 bit**。

SRAM（Static Random Access Memory）用一个**双稳态触发器**来存储数据。一个典型的 SRAM 单元由 6 个晶体管组成——两个交叉耦合的反相器构成一个"锁存器"，一旦写入就会一直保持状态，只要不掉电。这种结构的好处是**速度快**（读写都在 1~2 ns 量级）、**不需要刷新**。但代价也很明显：6 个晶体管占用的硅片面积很大，而且始终有静态功耗（反相器一直有电流流过）。所以 SRAM 适合用作 CPU 内部的 Cache（几十 KB 到几十 MB），但绝对不可能用来做 GB 级别的主存——成本和面积都不可接受。

DRAM（Dynamic Random Access Memory）则走了另一条路。它用一个**电容器**来存储数据——电容两端有电压差代表 1，没有电压差代表 0。核心思路是**用最少的元件存 1 个 bit**，从而在同样的硅片面积上塞进更多的存储单元。最终 DRAM 单元只用了 **1 个晶体管 + 1 个电容器**（所以叫 1T1C），比 SRAM 省了 6 倍的晶体管数。

但这是一个"魔鬼交易"——用面积换速度、换复杂度：
- 电容会**漏电**。存储在电容上的电荷经过几十毫秒就会流失殆尽，所以 DRAM 必须**定期刷新**——每隔几十微秒就要把所有存储单元读出来再写回去。这就是 DRAM 中 "D"（Dynamic，动态）的含义——数据是"动态保持"的，不像 SRAM 那样"静态保持"。
- 电容的电荷量极小（飞法级），读操作时 BL 上的电压变化只有 **几十到一百毫伏**——远小于数字电路能直接识别的逻辑电平。所以 DRAM 需要一个**高度灵敏的模拟放大器**（Sense Amplifier）来判别 0 和 1。

理解了这两个基本事实——**漏电需要刷新**、**信号微弱需要放大**——你就理解了 DRAM 所有设计的出发点。下面各节我们逐一展开。

## 2. 1T1C 单元：一个晶体管和一个电容的巧妙组合

### 2.1 物理结构

一个 DRAM 存储单元由三个部分组成：

```
         Word Line (WL) — 字线
              │
              │  Gate
    ┌─────────┴─────────┐
    │   NMOS Transistor  │  ← Access Transistor (存取晶体管)
    │   (开关)           │
    └─────────┬─────────┘
              │
              │  Drain
              ├────────── Storage Node (SN, 存储节点)
              │
         ─────┴─────
        ───  Cs  ───     ← Storage Capacitor (存储电容, 约 20-30 fF)
        ────────────
              │
            Vplate (≈ Vcc/2) — 电容下极板

         Bit Line (BL) ← 连接到 Source 端
```

我们先逐一理解每个部分：

**Access Transistor（存取晶体管）**是一个 NMOS 管。它的栅极（Gate）接到 Word Line（字线，WL），源极（Source）接到 Bit Line（位线，BL），漏极（Drain）接到存储节点（Storage Node，SN）。NMOS 管的特性很简单：当 Gate 电压高于 Source 电压一个阈值（Vth ≈ 0.7V）时，管子导通——电流可以在 Source 和 Drain 之间流动；当 Gate 电压低于阈值时，管子关断——Source 和 Drain 之间是高阻断路。

所以 WL 就是"开关控制线"：高电平 → 晶体管导通 → 电容和 BL 连通；低电平 → 晶体管关断 → 电容孤立、电荷被困在里面。存储电容 Cs 通常做在晶体管下方或旁边，它的上极板连接 Storage Node，下极板接到一个固定的偏置电压（通常是 Vcc/2，也就是电源电压的一半）。

**Storage Capacitor（存储电容）**的电容值大约在 **20~30 fF**（飞法，即 10⁻¹⁵ 法拉）。这个数字非常小——作为对比，你手指靠近一个金属焊盘产生的寄生电容可能就有几 pF（皮法，10⁻¹² 法拉），比 DRAM 电容大两个数量级。电容这么小的原因是：要在几十纳米的工艺节点上做几千万个单元，每个单元的面积必须极度压缩。电容值小而存储的电荷少，这就是后面所有挑战的根源。

### 2.2 写操作：如何把数据存入电容

写操作是最容易理解的：

- **写 "1"**：先把 BL 驱动到 Vcc（高电平），然后 WL 导通。这时 BL（Vcc）通过导通的 NMOS 管给 Cs 充电，电流从 BL 流向电容，直到 Cs 两端的电压也接近 Vcc。然后 WL 关断，电荷被"锁"在电容里。
- **写 "0"**：先把 BL 驱动到 0V（低电平），然后 WL 导通。这时 Cs 上如果有电荷，就会通过 NMOS 管放电到 BL，电流从电容流向 BL，直到 Cs 两端电压接近 0V。WL 关断，电容保持放电状态。

这里有一个容易被忽略但非常重要的细节：NMOS 管在导通时并不是一个理想的 0Ω 开关。它有一个有限的导通电阻 Ron（通常在几 kΩ 量级），所以充电和放电都需要一定的时间。这也是为什么 DDR 时序中有 tWR（Write Recovery Time）这个参数——如果电容还没来得及充满（或放完）就把 WL 关断了，写入的数据就可能处于一个"半高不低"的模糊状态，导致后续读取出错。

### 2.3 读操作：从电容"偷"出一点点信号

读操作比写操作复杂得多，也是 DRAM 最核心的设计挑战。它的基本流程是：

1. 先把 BL **预充电**到一个中间电平（Vcc/2）
2. WL 导通，让电容和 BL 发生**电荷共享**
3. 电容上的电荷会"稀释"到 BL 的寄生电容上，导致 BL 电压发生微小的上升或下降
4. 用 Sense Amplifier 把这个微小的电压变化**放大**到满摆幅

这里最关键的现象是**电荷共享**（Charge Sharing）：电容 Cs 上存的电荷是 Qs = Cs × Vs（Vs 是电容两端的电压）。当 WL 导通后，Cs 和 BL 的寄生电容 CBL 并联成一个更大的电容，总电荷 Qs 被分配到总电容 Cs + CBL 上。而 CBL 通常远大于 Cs——因为一根 BL 要连接几百到几千个存储单元，每个单元的 Drain 都会贡献一点点寄生电容，累积起来 CBL 大约在 **200~300 fF**，是 Cs 的十倍左右。

结果就是：读操作时 BL 上的电压变化非常微小。根据电荷守恒可以算出来：

```
       Cs                 25 fF
δV ≈ ────── × (Vs - VBL) ≈ ──────────── × (1.1V - 0.55V) ≈ 50 mV
     Cs + CBL             25 + 250 fF
```

**读一个存了 1.1V 电荷的电容，在 BL 上只产生了大约 50 mV 的变化。** 这就是 DRAM 读操作的物理现实——信号极端微弱，只有供电电压的不到 5%。

这个微小的电压差意味着两件事：第一，需要用非常灵敏的放大电路来判别 0 和 1；第二，读操作从电容中偷走了电荷（这些电荷被稀释到大电容 CBL 上了），所以**读操作是破坏性的**——读完之后电容里的数据已经被破坏了，必须再写回去。这种"读后回写"的需求贯穿了 DRAM 设计的方方面面。

## 3. 从单端到差分：为什么需要两条 Bit Line

读操作只产生 ~50mV 的电压变化，这是一个很尴尬的量级。如果用单端方案（一根 BL 和一个固定的参考电压比较），这个 50mV 很容易被各种噪声淹没——电源纹波可能有几十 mV，温度漂移也可能有几十 mV，旁边的 BL 翻转还会通过寄生电容耦合过来（串扰）。

DRAM 工程师用了一个非常巧妙的办法来解决这个问题：**差分信号**。

DRAM 中每一列实际上有**两根 Bit Line**：一根叫 BL（正线），一根叫 /BL（负线，读作"BL bar"）。两根线在物理上并排走线，经过的环境完全一样。在读操作之前，两根线都被预充电到**完全相同的电压**——Vcc/2。然后：

- 如果要读的 Cell 存的是 **1**（Cs 上有电荷）：WL 导通后，Cs 向 BL 注入电荷 → **BL 电压微升**（比如从 Vcc/2 升到 Vcc/2 + 50mV）。而 /BL 没有任何 Cell 连接（它对应的 WL 是关断的）→ /BL 保持在 Vcc/2。
- 如果要读的 Cell 存的是 **0**（Cs 上无电荷）：WL 导通后，BL 上的电荷向 Cs 流入 → **BL 电压微降**（比如从 Vcc/2 降到 Vcc/2 - 50mV）。/BL 同样保持在 Vcc/2。

于是我们就得到了一个差分电压：BL 和 /BL 之间要么有 +50mV 的差（代表 1），要么有 -50mV 的差（代表 0）。

差分方案的精妙之处在于**共模抑制**：如果电源有一个 30mV 的噪声毛刺，它同时对 BL 和 /BL 产生同样的扰动——两根线一起上升 30mV，它们之间的差值不变。如果温度变化导致晶体管阈值漂移，两根线上连接的 SA 电路同时受影响——差分差值依然不变。

> 这个思想跟 USB、HDMI、PCIe 用差分信号抗干扰的原理完全一样，只不过这里的"信号"不是 GHz 的高速数据流，而是一个只有 50mV 的**直流电压差**。

## 4. 差分 Sense Amplifier 的完整电路结构

现在我们知道每一列有 BL 和 /BL 两根线，需要有一个电路能把这 50mV 的差分信号放大到 Vcc 和 GND 的全摆幅。这个电路就是 **Sense Amplifier（感知放大器，简称 SA）**，它是整个 DRAM 芯片中**最重要的模拟电路**。

### 4.1 整体结构

SA 由三个子电路组成：**EQ（均衡电路）**、**SAN（N 型感知放大器）**、**SAP（P 型感知放大器）**，以及两个辅助信号——**CSL（列选通线）**和**WE（写使能）**。

```
                    Vcc
                     │
               ┌─────┴─────┐
               │    SAP    │  ← PMOS 交叉耦合对 (负责上拉)
               │           │
               ├─┬───┬───┬─┤
               │ │   │   │ │
          BL ──┼─┤   ├───┼─┼── /BL
               │ │   │   │ │
               ├─┴───┴───┴─┤
               │    SAN    │  ← NMOS 交叉耦合对 (负责下拉)
               │           │
               └─────┬─────┘
                     │
                    GND

    辅助电路 (每列):
    ┌────────────────────────────┐
    │  EQ   — 均衡电路 (3 NMOS)  │ ← Precharge 时将 BL 和 /BL 短接并拉到 Vref
    │  CSL  — 列选通信号         │ ← 将 SA 输出连接到全局 Data Bus
    │  WE   — 写使能信号         │ ← 写操作时强制驱动 BL 对
    └────────────────────────────┘
```

### 4.2 SAN：NMOS 交叉耦合对（下拉侧）

SAN 由两个 NMOS 管组成，它们的连接方式非常特别：**管子 A 的 Drain 接 BL、Gate 接 /BL；管子 B 的 Drain 接 /BL、Gate 接 BL**。这种"交叉耦合"的结构是正反馈的核心。

SAN 的工作过程如下：两个 NMOS 管的 Source 都接到一个叫 **SAN** 的控制信号上（不是直接接地）。平时 SAN 信号保持在高电平或浮空——两个 NMOS 管都不导通，SA 处于休眠状态。当需要放大时，SAN 信号被**驱动到低电平（GND）**。

此时两个 NMOS 管开始"竞争"：谁的 Gate 电压高，谁就导通得更多。假设读 1 时 BL 比 /BL 高 50mV——这意味着 BL 侧 NMOS 的 Gate（接 /BL，较低）得到的电压比 /BL 侧 NMOS 的 Gate（接 BL，较高）低。结果 /BL 侧的 NMOS 导通更强，把 /BL 更快地拉向 GND。而 /BL 一降低，BL 侧 NMOS 的 Gate 就更低，导通更弱，BL 就更不容易被拉低。这是一个**正反馈循环**——微小的初始差异被不断放大，最终 /BL 被拉到了 GND，而 BL 几乎不动（维持在 Vcc/2 附近）。

> 正反馈是模拟电路中实现高增益的经典手段。这里的交叉耦合 NMOS 对就像一个"模拟触发器"——它会自动倒向差异最大的状态。

### 4.3 SAP：PMOS 交叉耦合对（上拉侧）

SAN 只做了一半的放大——它把低侧拉到了 GND，但高侧还停留在 Vcc/2 左右。要得到完整的数字电平，还需要 SAP 来完成高侧的拉升。

SAP 的结构是 SAN 的"镜像"——用两个 PMOS 管交叉耦合，Source 接到一个叫 **SAP** 的控制信号上。平时 SAP 保持在低电平或浮空。在 SAN 完成下拉后（/BL ≈ GND），SAP 信号**被驱动到高电平（Vcc）**。

此时两个 PMOS 管开始竞争：PMOS 管的特性是 Gate 电压越低导通越强。由于 /BL 已经被拉到了 GND，它的 Gate（接 BL，此时还在 Vcc/2 附近）比另一侧的 Gate（接 /BL ≈ GND）高——所以 /BL 侧的 PMOS 导通更弱（Gate 电压高 = 更难导通），而 BL 侧的 PMOS 导通更强（Gate 接 /BL ≈ GND = 最低，最容易导通）。BL 被迅速拉向 Vcc。这又是一个正反馈——BL 越高 → /BL 侧 PMOS 的 Gate 越高 → /BL 越难被上拉 → BL 越容易被上拉到 Vcc。

**最终结果**：BL = Vcc（强逻辑 1），/BL = GND（强逻辑 0）。起始时只有 50mV 的差异，经过 SAN 和 SAP 两级正反馈放大，变成了 Vcc 和 GND 的全摆幅。这个放大过程通常只需要几纳秒。

### 4.4 EQ：均衡电路

EQ（Equalization，均衡电路）由 3 个 NMOS 管组成。它们的任务非常简单但至关重要：在每次读写操作开始之前，把 BL 和 /BL 短接在一起，并且都拉到 Vref（Vcc/2）。

为什么需要这一步？因为上一次操作结束后，BL 可能停在 Vcc 而 /BL 停在 GND（如果上次读的是 1），或者反过来。如果不先把它们均衡到同一个起点，下一次读操作时 Access 阶段产生的 50mV 差异就会叠加在 1.1V 的残留偏置上——信号完全被淹没了。

EQ 信号在 Precharge 阶段有效，三个 NMOS 管同时导通：外侧两个管子分别把 BL 和 /BL 连到 Vref，中间一个管子把 BL 和 /BL 短接。由于 NMOS 管的导通电阻很小，BL 和 /BL 在非常短的时间内（几百皮秒到几纳秒）就被均衡到了 Vref。然后 EQ 关断，为接下来的 Access 阶段做好准备。

### 4.5 CSL 和 WE：连接外部世界的桥梁

SA 把 BL 对上的电压放大到了 Vcc/GND，但这个结果还需要传输到芯片外部的数据总线上（或者反过来，写数据需要从外部写入 BL 对）。这就是 **CSL（Column Select Line，列选通线）**和 **WE（Write Enable，写使能）**的作用。

CSL 由 Column Decoder 控制。当某一列的 CSL 有效时，该列的 SA 输出通过一对选通管连接到全局的 Data Bus。在读操作中，SA 已经锁存了 BL 对的值，CSL 导通后这个值就被读到 Data Bus 上。在写操作中，CSL 和 WE 同时有效，外部数据总线直接驱动 BL 对——这相当于"强制覆盖"SA 当前锁存的值，把新数据写入电容。

## 5. Vref = Vcc/2：这个看似简单的选择有多重要

你可能已经注意到，Vref（预充电参考电压）总是被设为 **Vcc/2**（电源电压的一半）。这不是一个随意的选择——它背后有多重考量：

**第一，对称性。** 读 1 时 BL 从 Vcc/2 往上偏，读 0 时 BL 从 Vcc/2 往下偏。两个方向上的电压变化幅度是对称的，SA 感知两个方向的灵敏度一样。如果 Vref 设在 Vcc/4，那么读 0 时 BL 只能往下偏 Vcc/4（就到地了），读 1 时可以往上偏 3Vcc/4——不对称，SA 的设计会非常困难。

**第二，噪声裕度最大化。** 数字电路中"0"和"1"的判断阈值通常也在 Vcc/2 附近。BL 从 Vcc/2 出发，意味着不管是读 0 还是读 1，信号偏离阈值的方向和幅度都是最优的。

**第三，功耗。** Precharge 时需要把 BL 对从之前的 Vcc/GND 状态充电/放电到 Vref。如果 Vref = Vcc/2，那么不管之前的状态是什么，充放电的摆幅最大只有 Vcc/2。如果 Vref = Vcc，那每次读 0 后 Precharge 需要把 BL 从 GND 拉到 Vcc——功耗翻倍。

**第四，速度。** BL 从 Vcc/2 往任一方向摆动，达到 SA 可感知差分电压需要的时间最短。因为 BL 的 RC 时间常数是固定的，摆幅越小就越快。

## 6. 读操作的四个阶段（上）：Precharge 和 Access

现在我们把前面学到的所有知识串联起来，完整走一遍 DRAM 的读操作流程。读操作分为四个阶段：**Precharge（预充电）→ Access（访问）→ Sense（感知）→ Restore（恢复）**。这一节我们先讲前两个阶段。

### 6.1 阶段一：Precharge（预充电）

在 DRAM 芯片上电或完成一次操作之后，存储阵列处于**空闲状态**——没有 Word Line 被激活，所有 Sense Amplifier 处于休眠状态。此时各列的 BL 和 /BL 可能残留着上次操作结束时的电压（Vcc 或 GND），也可能处于不确定的浮空态。Precharge 阶段的目标就是把所有列的 BL 和 /BL 统一初始化到 Vref（Vcc/2），为下一次操作做好准备。

操作步骤：
1. **EQ 信号使能**。所有列的 EQ 电路同时激活——三个 NMOS 管导通，把 BL 和 /BL 短接并拉到 Vref。
2. **等待 BL 对稳定**。BL 和 /BL 的寄生电容 CBL 通过 EQ 管的导通电阻充放电，电压迅速收敛到 Vref。这个过程的时间由 RC 时间常数决定，通常在几百皮秒到几纳秒。
3. **EQ 信号关闭**。BL 和 /BL 都稳定在 Vref，等待 Access 阶段。

这个阶段对应 DDR 时序中的 **tRP（Row Precharge Time）**。tRP 规定了从 PRE 命令发出到下一个 ACT 命令之间的最小间隔。如果间隔太短，BL 对还没来得及充分均衡，残留的电压偏置就会干扰下一次读操作的差分感知。

### 6.2 阶段二：Access（访问）

Precharge 完成后，BL 和 /BL 稳定在 Vref。接下来就是"开门"的时刻了。

操作步骤：
1. **Row Decoder 选中一条 Word Line**，将其驱动到**高电平**。这个高电平通常比 Vcc 还要高一些（因为 NMOS 管的阈值损失问题，通常在 Vcc + Vth 以上，由 VPP 升压电路提供），以确保 Access Transistor 能充分导通，把电容上 Vcc 的电压无损失地传到 BL。

2. **整行所有存储单元的 Access Transistor 同时导通**。这一行有多少列，就有多少个 Cell 的电容同时连接到各自的 BL（或 /BL）。注意：对于每一列，只有一个 Cell 被连接到 BL 或 /BL 中的一根——另一根保持连接在 Vref 上（它的 WL 是关断的）。具体接哪一根取决于这一列的 Cell 是接在 BL 上还是 /BL 上。

3. **电荷共享发生**。每个 Cell 电容 Cs 和对应的 BL 电容 CBL 并联——Cs 上的电荷被"稀释"到大电容上。BL 的电压变化由我们之前推导的电荷守恒公式决定。

4. **/BL 保持不动**。因为 /BL 一侧没有 Cell 被激活（对应的 WL 是关断的），/BL 始终稳定在 Vref，作为差分比较的基准。

此时每一列都有了一对差分电压：如果 Cell 存的是 1，BL 比 /BL 高 ~50mV；如果 Cell 存的是 0，BL 比 /BL 低 ~50mV。但 50mV 还远不是数字电平——接下来需要 Sense Amplifier 出场。

---

## 7. 读操作的四个阶段（下）：Sense 和 Restore

### 7.1 阶段三：Sense（感知放大）

当 Access 阶段产生的差分电压建立后（通常需要几百皮秒到几纳秒），SA 开始工作。Sense 阶段分两步走：

**第一步：SAN 激活（下拉）**

SAN 控制信号被驱动到 GND。以读 1 为例（BL 比 /BL 高 50mV）：
- /BL 侧的 NMOS 管：Gate = BL（比 /BL 高 50mV）→ 导通更强
- BL 侧的 NMOS 管：Gate = /BL（比 BL 低 50mV）→ 导通更弱

/BL 侧的 NMOS 先把 /BL 往下拉 → /BL 更低了 → 导致 BL 侧的 NMOS 的 Gate 更低 → BL 侧 NMOS 更不容易导通 → BL 几乎不被往下拉。正反馈不断强化这个趋势：/BL → GND，BL 保持在 Vcc/2 附近。

**第二步：SAP 激活（上拉）**

在 SAN 已经建立了 BL 和 /BL 之间的显著差异后（/BL 接近 GND），SAP 控制信号被驱动到 Vcc。此时：
- BL 侧的 PMOS：Gate = /BL（接近 GND）→ 导通非常强 → BL 被拉向 Vcc
- /BL 侧的 PMOS：Gate = BL（在 Vcc/2 以上）→ 导通很弱 → /BL 不会被拉高

正反馈再次起作用：BL 被拉得越高 → /BL 侧的 PMOS 的 Gate 越高（越难导通）→ /BL 越不受影响 → BL 更快上升到 Vcc。

**最终状态**：BL = Vcc（全逻辑 1），/BL = GND（全逻辑 0）。

这里有一个时序细节值得注意：为什么是先拉 SAN 再拉 SAP，而不是同时拉？因为如果同时拉，PMOS 和 NMOS 可能同时导通形成"贯通电流"（shoot-through current），从 Vcc 直接流到 GND，产生很大的功耗尖峰。分步操作可以避免这个问题。

### 7.2 阶段四：Restore（恢复/回写）

Sense 阶段结束后，BL = Vcc，/BL = GND。此时 WL 仍然是高电平，Access Transistor 仍然导通。这意味着存储电容 Cs 直接面对着 BL 上的 Vcc 电压（如果读的是 1）或 GND 电压（如果读的是 0）。

**Restore 就是利用这个窗口把数据写回电容的过程。** 因为我们之前在 Access 阶段从电容中偷走了电荷（通过电荷共享），Cs 现在处于一个"半空"的状态——它的电压既不是 Vcc 也不是 GND，而是介于两者之间的某个模糊值。如果不写回去，这个数据就永远丢失了。

Restore 的操作非常简单：保持 WL 高电平足够长的时间，让 Cs 通过 Access Transistor 从 BL（Vcc）充电或向 BL（GND）放电，直到 Cs 两端的电压恢复到 Vcc（存 1）或 GND（存 0）。这个充放电需要的时间由 Access Transistor 的导通电阻 Ron 和 Cs 的乘积（RC 时间常数）决定。

Restore 完成后，WL 被驱动到低电平，Access Transistor 关断，Cs 重新被隔离——数据恢复完毕。

**这就是"读即刷新"的物理本质**：读一行数据的过程（Access → Sense → Restore），天然地完成了对这一行所有 Cell 的刷新。你不需要额外地"刷新"刚刚读过的那一行——因为读操作本身已经做了 Restore。

---

## 8. 写操作：为什么比读操作多一步

写操作的前半程——Precharge、Access、Sense——与读操作完全一样。为什么要先 Sense？因为写操作通常只修改一行中的部分列，其他列的数据需要保留。而 Access 阶段是破坏性的——所有列的数据都被电荷共享稀释了。所以必须先走完 Sense 把整行数据锁存到 SA 中，再把要修改的那些列用新数据覆盖。

写操作的关键是 **Write Recovery** 阶段——它发生在 Restore 结束之后，是写操作独有的第五个阶段：

1. **CSL（列选通信号）有效**。Column Decoder 选中了要写入的那些列，对应的 SA 输出通过选通管连接到全局 Data Bus。
2. **WE（写使能）有效**。Data Bus 上的写数据通过一个**强驱动电路**强制施加到被选中列的 BL 对上——直接覆盖 SA 当前锁存的值。这个驱动电路必须比 SA 的锁存能力更强，才能把 BL 对"翻转"过来。
3. WL 仍然保持高电平，新的 BL 电压通过 Access Transistor 对 Cs 充电或放电，将新数据写入电容。
4. 写入完成后，WL 关断，Cs 隔离。

**为什么需要 tWR（Write Recovery Time）？** 因为 Step 3 需要时间。如果 WE 刚把 BL 翻过来就关 WL，Cs 可能还没来得及完全充/放电，新数据只写入了一半。tWR 保证写数据有足够的时间从 BL 传到 Cs 并稳定下来。这就是 DDR 时序中 tWR 参数背后的物理原因。

## 9. 从 Cell 到 Array：如何组织几千万个单元

一个 8Gb 的 DDR4 芯片有大约 80 亿个 1T1C 单元。它们不是杂乱无章地堆在一起，而是按照一个非常规整的**行列矩阵**来组织的。

### 9.1 Memory Array 的基本结构

一个 Memory Array 就是一个二维网格：

```
           Col 0      Col 1      Col 2     ...     Col M-1
           ┌───┐      ┌───┐      ┌───┐             ┌───┐
  BL[0]    │   │      │   │      │   │             │   │
 /BL[0]    │   │      │   │      │   │             │   │
           │   │      │   │      │   │             │   │
WL[0] ─────┼───┼──────┼───┼──────┼───┼─────────────┼───┼── Row 0
           │   │      │   │      │   │             │   │
WL[1] ─────┼───┼──────┼───┼──────┼───┼─────────────┼───┼── Row 1
           │   │      │   │      │   │             │   │
 ...       └───┘      └───┘      └───┘             └───┘
           SA[0]      SA[1]      SA[2]           SA[M-1]
             │          │          │                 │
             └──────────┴────┬─────┴─────────────────┘
                             │
                     Column Decoder
                     (CSL 选择)
                             │
                        Data Bus
```

在这个矩阵中：
- **同一行的所有 Cell 共享同一根 Word Line**。当这根 WL 被激活时，整行的所有 Cell 同时连接到各自的 BL（或 /BL）。这就是为什么 ACT 命令的粒度是"行"——一个 ACT 命令打开的是整个一行，不是单个 Cell。
- **同一列的所有 Cell 共享同一对 Bit Line（BL + /BL）和同一个 Sense Amplifier**。这意味着同一列上的不同 Cell 不能同时被访问——它们的电容都挂在同一对 BL 上，同时打开会造成冲突。这就是为什么同一 Bank 同一时间只能有一个 Row 处于打开状态。
- **每列配有一个独立的 Sense Amplifier**。所以当你激活一行时，这一行有多少列，就有多少个 SA 同时工作——数据被并行地读取到 SA 阵列中。

### 9.2 为什么用矩阵而不是其他结构

矩阵组织有几个关键优势：

**第一，最大化共享**。如果一个阵列有 R 行 × C 列，只需要 R 根 WL + C 对 BL + C 个 SA，而不是 R × C 套独立的读写电路。这极大节省了面积。

**第二，批量操作**。一次 ACT 可以同时激活整行所有列——这意味着内部带宽可以非常高。对于 C = 2048 列的阵列，一次 ACT 就在 SA 阵列中"缓存"了 2048 bits = 256 Bytes 的数据。后续只需通过 Column Decoder 从中选取需要的列输出即可。

**第三，方便刷新**。刷新操作只需要依次激活每一行——SA 自动完成 Sense 和 Restore。不需要逐 Cell 做。

## 10. 行多列少的深层原因：寄生电容的权衡

你可能注意到了：DRAM 阵列中行数（WL 数量）通常远多于列数（BL 数量）。一个典型的数组可能有 4096 行但只有 1024 列（或者 2048 列）。为什么不是反过来，或者方的？

原因在于**Bit Line 的寄生电容 CBL**。每根 BL 要连接到这一列上**所有**的 Cell——每一个 Cell 的 Access Transistor Drain 都会贡献一点寄生电容。行数越多 → 一根 BL 上挂的 Cell 越多 → CBL 越大。而 CBL 越大，读操作时电荷共享产生的 δV 就越小——因为 δV 与 Cs/CBL 成正比。

所以有一个基本的物理权衡：**行数受限于可接受的 δV**。如果 δV 太小（比如 < 30mV），SA 就无法可靠地判别，误码率会急剧升高。工艺节点每前进一代，Cs 变得更小（密度更高），CBL 也相应要控制——这就是为什么每代 DRAM 的阵列组织都需要精心优化。

列数同样有限制，但维度不同。每增加一列 = 增加一对 BL + 一个 SA → 面积增加。所以列的规模受限于面积预算。

---

## 11. Data Width：x4 / x8 / x16 是怎么实现的

现在我们知道一个 Memory Array 每次读写输出 1 bit。那 x4、x8、x16 这些接口宽度是怎么来的？

答案出乎意料地简单：**并联多个相同的 Memory Array**。

- **x4 器件**：4 个 Memory Array 并联。每个 Array 负责 1 bit（DQ[0] ~ DQ[3]）。4 个 Array 共享相同的地址（Row、Column、Bank）和命令。
- **x8 器件**：8 个 Array 并联 → DQ[0] ~ DQ[7]。
- **x16 器件**：16 个 Array 并联 → DQ[0] ~ DQ[15]。

所有并联的 Array 执行完全相同的操作——ACT 时同时打开各自的同一行，READ 时同时从各自的同一列读取数据。它们只是把数据分散到了不同的 DQ pin 上。

> 这意味着一个 Bank 不是"一个 Array"，而是 **data_width 个 Array 的集合**。Bank 中的所有 Array 共享一套行列地址线和命令总线，并行工作，从外部看就像是一个更宽的存储器。

---

## 12. 层级架构：Cell → Bank → Bank Group → Rank → DIMM

现在我们可以自底向上地把整个层级串起来。这是理解 DDR 架构最重要的框架。

### 12.1 第一层：Cell

一个 1T1C 单元，1 bit。几千万到几亿个 Cell 组成一个 Memory Array。

### 12.2 第二层：Memory Array

Row × Column 个 Cell 组成的矩阵，加上 Row Decoder、Column Decoder 和 SA 阵列。一次 ACT 激活一整行到 SA 中。

### 12.3 第三层：Bank

**Bank = data_width 个 Memory Array 并联**。对于 x8 器件，一个 Bank 包含 8 个相同的 Array。Bank 拥有独立的 Row 地址和 Column 地址空间——不同 Bank 之间的操作完全独立。

Bank 是整个 DDR 系统中**并行性的基本单位**。控制器的 Bank 交错调度，本质上就是利用不同 Bank 的独立状态来隐藏单个 Bank 的 Precharge/Activate 开销。

### 12.4 第四层：Bank Group

Bank Group 是 DDR4 引入、DDR5 扩展的概念。多个 Bank 共享局部 IO 资源（比如数据通路），被编入同一个 Group。

Bank Group 的重要性体现在时序上：**不同 Bank Group 的 Bank 之间切换更快**（tCCD_S 更短），而同一 Bank Group 内的不同 Bank 之间切换更慢（tCCD_L 更长）。这是因为跨 Group 访问可以使用不同的 IO 通路，而 Group 内需要复用同一套 IO。

DDR4：4 BG × 4 Bank = 16 Banks
DDR5：8 BG × 4 Bank = 32 Banks

### 12.5 第五层：Rank

一个 Rank 是一组共享同一套 CA 和数据总线的 DRAM 芯片的集合。比如 CPU 的数据总线是 64-bit，如果用 x8 的 DRAM 芯片，就需要 8 颗芯片并联——这 8 颗芯片组成一个 Rank。它们总是同时被选中、执行相同的命令。

多个 Rank 可以共存于同一条 DIMM 上（比如双 Rank DIMM），通过不同的 CS_n（Chip Select）信号来选择当前操作的是哪个 Rank。不同 Rank 之间可以交错——Rank 0 正在忙于刷新时，Rank 1 可以继续提供数据。

### 12.6 第六层：DIMM

DIMM（Dual In-line Memory Module）就是我们俗称的"内存条"。它是将 Rank（一个或多个）和 DDR5 的 PMIC（电源管理芯片）集成在一块 PCB 上的完整内存模组。DDR5 DIMM 包含两个独立的 32-bit 子通道（Channel A 和 Channel B），每个通道有自己独立的 CA 总线和 Bank 阵列。

---

## 13. 为什么需要 Bank 交错：一条流水线的故事

理解了前面的层级结构，我们自然会产生一个问题：为什么要搞这么多 Bank？为什么不把所有存储单元堆成一个大 Bank 了事？

为了回答这个问题，我们先看看**只有一个 Bank 会怎样**：

假设你要连续读同一 Bank 的不同 Row：
1. ACT Row X → 等待 tRCD → READ → 数据出来 → 等待 tRTP → PRE → 等待 tRP
2. ACT Row Y → 等待 tRCD → READ → ...

在两次读取之间，你需要经历 PRE（让旧行回写并关闭）和 tRP（等 BL 均衡到位），再加上 ACT（开新行）和 tRCD（等 SA 稳定）。这一整套下来——ACT → READ → PRE → ACT → READ——叫做一个 **Row Cycle（tRC）**。对于 DDR5-4800，tRC ≈ 46 ns。在这 46 ns 中，数据总线只在中间的极短时间（BL16 = 约 3.3 ns）有数据传输。

**总线利用率 = 3.3 / 46 ≈ 7%**。93% 的时间总线是空的，数据脚在"等"DRAM 内部忙完。

这就是"只有一个 Bank"的灾难——你无法隐藏内部的预充电和激活开销，因为它们是串行的。

### Bank 交错的解决方案

现在有了多个 Bank。当 Bank 0 正在忙着 PRE → ACT 的时候，你可以去访问 Bank 1（它已经处于 ACTIVE 状态，Row 开着等着你读）。当 Bank 1 的数据传输完去做 PRE 时，你又可以去访问刚刚 ACT 完的 Bank 2。这就是**Bank 交错（Bank Interleaving）**——用多个独立 Bank 的并行操作来填满总线。

理想情况下，如果你有足够多的 Bank，且访问分布均匀，你可以让总线几乎 100% 的时间都在传输数据。每个 Bank 的内部开销（tRCD、tRP、tRC）被"隐藏"在其他 Bank 的操作背后。这也是为什么 DDR5 要把 Bank 数量从 16 个（DDR4）增加到 32 个——Bank 越多，调度空间越大，隐藏开销的能力越强。

### Bank Group 的进一步优化

Bank Group 在 Bank 的基础上增加了一层优化。同一 BG 内的 Bank 共享 IO 通路，切换时需要更多时间（tCCD_L）；跨 BG 的 Bank 有独立 IO 通路，切换更快（tCCD_S）。控制器的调度器可以利用这一点：优先把连续的命令派发到不同 BG 的 Bank 上，以享受更短的命令间隔。

## 14. 地址解码：如何在海量单元中精确定位

要对一个特定的存储单元进行读写，我们需要知道它所在的 Bank Group、Bank、Row、Column。这些信息通过地址总线传入 DRAM，由解码器转换成具体的选通信号。

### 14.1 Row Decoder（行解码器）

Row Decoder 的任务是：根据 Row Address（比如 14 位），从 2^14 = 16384 根 Word Line 中选中一根，将其驱动到高电平。

它通常分两级：第一级是 Pre-decoder（预解码器），将地址分成几段分别做初步译码，减少后级所需晶体管的数量；第二级是 Word Line Driver（字线驱动器），提供驱动 WL 的强大电流——因为一根 WL 要同时驱动整行几千个 Cell 的 Access Transistor Gate，负载电容很大。

### 14.2 Column Decoder（列解码器）

Column Decoder 的任务是：根据 Column Address，选通相应的 CSL（Column Select Line），将对应列的 SA 输出连接到全局 Data Bus。

由于一个阵列可能有几千列，而数据总线宽度只有几到几十位，Column Decoder 通常是**多级选择**的——先选一组列（比如 8 列一组），再从组内选具体的位。这种分级选择减少了全局 Data Bus 的连线数量。

### 14.3 Bank / BG Decoder

Bank Address（通常 2~3 bit）和 Bank Group Address（DDR5 是 3 bit）通过独立的解码路径，选择目标 Bank Group 和 Bank。选中后，该 Bank 的 Row Decoder 和 Column Decoder 被激活，其他 Bank 保持休眠。

### 14.4 地址分时复用（DDR4 及之前）

在 DDR4 及之前的标准中，**行地址和列地址共享同一组物理引脚** A[17:0]，通过不同的命令来区分当前传输的是行地址还是列地址。ACT 命令时，地址总线上的值是 Row Address；READ/WRITE 命令时，地址总线上的值是 Column Address。这种"分时复用"方式可以节省一半的地址引脚。

DDR5 改变了这种做法：采用 **CA[13:0] 统一命令地址总线**，命令和地址在同一个 14-bit 总线上编码传输，不再需要独立的 RAS/CAS/WE 信号。这是 DDR5 信号接口的一个重大变化。

## 15. 刷新：DRAM 最独特的约束

### 15.1 为什么必须刷新？

回到最根本的物理事实：电容会漏电。

存储在 Cs 上的电荷通过以下途径流失：
- **PN 结漏电流（主要）**：Access Transistor 的 Drain-Body 二极管即使在反向偏置下也有微弱的漏电流（pA~nA 级）。这是电荷流失的最主要途径。
- **亚阈值漏电**：当 WL 为低电平时，Access Transistor 应该完全关断，但实际上在深亚微米工艺中，晶体管即使"关断"也有一条微弱的亚阈值电流通道。栅极长度越短，亚阈值漏电越严重。
- **电容介质漏电**：电容的介质材料不是完美绝缘体，有极小的漏电流。

综合这些效应，一个典型 DRAM 单元的电荷保持时间大约是 **64ms ~ 几百 ms**（取决于温度和工艺）。为了保证数据可靠，JEDEC 标准规定：**所有 Cell 必须在 64ms（Tcase ≤ 85°C）或 32ms（高温模式）内至少被刷新一次**。

### 15.2 刷新操作 = 伪读操作

刷新操作本质上就是一次"简化的读操作"：

1. 激活一行（WL 导通）→ Access 阶段
2. SA 自动感知并放大 → Sense 阶段
3. SA 自动回写 → Restore 阶段
4. 关闭 WL → 内部自动 Precharge

整个过程中，CSL 始终不导通——数据不需要输出到 Data Bus。DRAM 内部有一个**刷新计数器**，自动递增 Row 地址，保证所有行在 64ms 内被循环刷新一遍。外部控制器只需要定期发送 REF 命令即可。

对于 DDR5，刷新有两种模式：
- **REFab（Refresh All Banks）**：所有 Bank 同时各刷新一行，死区时间 = tRFCab（~295 ns）
- **REFsb（Refresh Same Bank）**：只刷新指定 Bank Group 的一个 Bank，死区时间 = tRFCsb（~120 ns，短得多）

DDR5 推荐使用 REFsb 来实现"碎片化刷新"——把大块的刷新死区分散到不同的时间点，减少对正常访问的阻塞。

### 15.3 tREFI 的含义

tREFI（Refresh Interval）规定了控制器发 REF 命令的**平均间隔**。对于 DDR5-4800 的 1x 模式（正常温度），tREFI = 3.9 μs。这意味着控制器平均每 3.9 μs 就要发一次 REF 命令。注意 DDR5 的 tREFI 是 DDR4 的一半（3.9 vs 7.8 μs）——因为 DDR5 的密度更大，同一时间内需要刷新更多的行。

## 16. 通信接口的演进：从 RAS/CAS 到 CA 总线

DRAM 的接口设计一直在演进。理解这个演进过程，有助于理解为什么 DDR5 的接口长成现在这个样子。

### 16.1 传统接口（SDRAM ~ DDR4）

DDR4 及之前的标准使用**独立控制信号**的方式来编码命令：RAS_n、CAS_n、WE_n 三根线（加上 DDR4 的 ACT_n），通过不同的组合来表示不同的命令。地址则通过 A[17:0] 引脚在 ACT 和 READ/WRITE 时分别传输。

这个设计的优点是**直观**——控制信号直接对应操作含义，硬件解码简单。但缺点是每个控制信号都要占用一个引脚，而且地址的"分时复用"使得寻址需要两个步骤。

### 16.2 DDR5 的 CA 总线

DDR5 改为 **CA[13:0] 统一命令地址总线**。这是一个 14-bit 宽的总线，在 CK 上升沿采样。大多数命令需要 2 个 CK 周期完成编码（第一个上升沿传命令码，第二个上升沿传附加数据），少数简单命令只需 1 个周期。

这种设计把命令和地址完全融合在一条总线上，减少引脚数的同时提供了更大的编码空间——DDR5 可以支持比 DDR4 多很多的命令变体（REFsb、SAME-BG-P、MPC 等），这些都需要额外的编码空间。

## 17. Prefetch：DDR 每一代提速的秘密武器

### 17.1 核心矛盾

DRAM 内部阵列有天然的速度瓶颈。Sense Amplifier 需要时间来感知和放大微弱信号（决定了 tRCD），Precharge 需要时间来均衡 BL 对（决定了 tRP）。这些模拟电路的时序约束使得内部阵列的频率只能做到 **200~400 MHz**。

但市场对接口速率的要求一直在指数增长——从 DDR-400 到 DDR5-6400，速率翻了 16 倍。内部频率跟不上怎么办？

### 17.2 Prefetch 方案

答案是用**面积换速度**——内部以低频宽位宽操作，一次取出多位数据，然后在输出端用高速串行化逻辑（SERDES）把它们逐个发出去。

这就是 Prefetch（预取）的核心思想：

| 代际 | Prefetch | 内部频率 (以接口速率为例) | 内部数据宽度 |
|------|----------|--------------------------|-------------|
| SDR | 1n | 100 MHz ≈ 100Mbps | 1 bit |
| DDR | 2n | 100 MHz → 200Mbps | 2 bits |
| DDR2 | 4n | 100 MHz → 400Mbps | 4 bits |
| DDR3 | 8n | 200 MHz → 1600Mbps | 8 bits |
| DDR4 | 8n | 300 MHz → 2400Mbps | 8 bits |
| DDR5 | 16n | 300 MHz → 4800Mbps | 16 bits |

以 DDR5-4800 为例，内部阵列以 **300 MHz** 的频率工作，一次从 Bank 中读出 **128 bits**（16n × 8-bit x8 器件），然后通过 8:1 的并转串逻辑，在 DQ 线上以 4800 MT/s 的速率串行输出——每个 DQ pin 在 8 个 CK 周期内输出 16 个数据沿（上升沿 + 下降沿）。

### 17.3 Prefetch 和 Burst Length 的关系

Burst Length（BL）恰好等于 Prefetch 数：BL16 = 16n Prefetch。这不是巧合——一次内部预取出来的数据，恰好通过一个 Burst 完整输出。这也意味着 BL 就是 DDR 的最小操作粒度：你不能只读预取数据的一半就停，因为内部是按 16n 的批次工作的。

### 17.4 为什么 DDR5 需要 16n

DDR4 已经是 8n Prefetch。DDR4-3200 需要内部阵列跑到 400 MHz，这对于 DRAM 工艺来说已经是极限了（SA 建立时间、Decoder 延迟等约束）。要进一步提速到 4800~6400 MT/s，8n 方案需要内部跑到 600~800 MHz——不可行。所以 DDR5 把 Prefetch 翻倍到 16n，让内部频率回到 ~300 MHz 这个舒适区间。

代价是 Burst Length 从 8 增加到 16，最小访问粒度变大（从 8B 变成 16B @ x8）。但对于现代 CPU 以 64B Cache Line 为单位的访问模式来说，这不是问题。

## 18. 完整芯片实例：Micron MT48LC32M4A2 拆解

现在让我们用前面学到的所有知识，来"解剖"一颗真实的 DRAM 芯片。我们选择 Micron 的 MT48LC32M4A2——一颗经典的 128Mb SDRAM。虽然它是一颗 SDR（不是 DDR），但它的内部阵列组织与 DDR3/DDR4/DDR5 一脉相承，非常适合用来理解芯片内部的物理结构。

### 18.1 芯片规格

| 参数 | 数值 |
|------|------|
| 容量 | 128 Mb = 16 MB |
| 组织 | 4 Banks × 4096 Rows × 2048 Columns × 4-bit |
| 供电 | 3.3V（SDR 时代） |
| 接口宽度 | x4（4 根 DQ） |

我们来拆解这组数字："4 Banks × 4096 Rows × 2048 Columns × 4-bit"。

### 18.2 一个 Bank 内部

每个 Bank 包含 4 个 Memory Array（因为是 x4 器件——DQ0~DQ3 各对应一个 Array）。每个 Array 的规模是 **4096 行 × 2048 列**。所以：
- **Row Decoder**: 12-bit 输入（2^12 = 4096）→ 选中 4096 根 WL 中的一根
- **SA 阵列**: 2048 个 SA（每列一个）→ 一次 ACT 激活整行 = 2048 bits 被锁存到 SA
- **Column Decoder**: 由于 Column Address 是 8-bit（2^8 = 256），但不是一一对应到列的——这里使用了分组选择。Column Decoder 输出 256 个 CSL 信号，每个 CSL 同时选通 **8 列**。所以 256 × 8 = 2048 列。列地址的低 3 位在第二级 MUX 中选择具体的一列。

**一个 Bank 的总容量** = 4096 × 2048 × 4 = 33,554,432 bits = 32 Mb。

**4 个 Bank 的总容量** = 32 × 4 = 128 Mb。验证了。

### 18.3 寻址过程实例

假设我们要读 **Bank 2, Row 1500, Column 1027**：

1. **ACT 命令**：
   - Bank Address = 2（BA[1:0] = 10）
   - Row Address = 1500 → Row Decoder → WL[1500] 被驱动到高电平
   - Bank 2 内 4 个 Array 的 WL[1500] 同时激活
   - 4 × 2048 = 8192 个 Cell 同时连接到各自的 BL → 8192 个 SA 开始工作
   - 经过 tRCD（Sense + Restore 完成），数据就绪

2. **READ 命令**：
   - Column Address = 1027
   - CSL 选择 = 1027 / 8 = 128 → CSL[128] 导通
   - 列内偏移 = 1027 % 8 = 3 → 第二级 MUX 选第 4 组
   - 选中的 SA 输出 → Data Bus → DQ[3:0] 输出

3. **PRE 命令**：WL[1500] 关断，BL 对均衡，SA 休眠。

### 18.4 控制逻辑

除了 Bank 阵列，芯片内部还有一个**Control Logic（控制逻辑）**模块。它包含一个命令解码器，把外部输入的 RAS_n/CAS_n/WE_n/CS_n 信号翻译成内部的时序控制信号——什么时候激活 WL、什么时候使能 SAN/SAP、什么时候选通 CSL、什么时候关断和均衡。外部 Memory Controller 发出的每一条命令，最终都是由这个控制逻辑翻译成精确的模拟时序来执行的。

## 19. 总结：从电容到 DDR5 的完整知识链

让我们把整篇文章的知识串成一条完整的链：

```
一颗电容 (1T1C Cell)
  │
  │ 问题: 读出的信号只有 ~50mV，噪声就能淹没它
  ▼
差分 Bit Line + Sense Amplifier
  │  Precharge → Access → Sense → Restore
  │  正反馈放大: 50mV → Vcc/GND 全摆幅
  │
  │ 问题: 怎么组织几千万个 Cell？
  ▼
Memory Array (Row × Column 矩阵)
  │  一次 ACT = 打开一整行到 SA 阵列
  │  Row Decoder + Column Decoder 精准定位
  │
  │ 问题: 怎么匹配接口位宽？
  ▼
Bank (data_width 个 Array 并联)
  │  x4/x8/x16 = 4/8/16 个 Array 并行工作
  │
  │ 问题: 单 Bank 的 PRE/ACT 开销太大，总线空闲 >90%
  ▼
多 Bank 交错 (Bank Interleaving)
  │  用 Bank 并行性隐藏 tRC 死区
  │  DDR4 16 Banks → DDR5 32 Banks
  │
  │ 问题: 不同 Bank 间 IO 切换也有开销
  ▼
Bank Group (共享 IO 的 Bank 编组)
  │  同 BG: tCCD_L (慢)，跨 BG: tCCD_S (快)
  │
  │ 问题: 一颗芯片的位宽不够
  ▼
Rank (多芯片并联匹配 64bit CPU 总线)
  │
  │ 问题: 内部阵列频率跟不上接口速率
  ▼
Prefetch (1n → 2n → 4n → 8n → 16n)
  │  内部慢、宽 → 接口快、窄 → 用面积换速度
  │
  │ 问题: 高速下信号完整性退化
  ▼
DDR5 特化: DFE均衡、Die内ECC、独立PMIC、双通道架构
```

### 五个关键认知

1. **读操作是破坏性的。** Restore 必须完成。刷新是 DRAM 的生命线。tRAS 这个参数存在的根本原因就是 Restore 需要时间。

2. **Sense Amplifier 是模拟电路，不是数字电路。** 它处理的是 mV 级的微弱信号，通过正反馈放大到数字电平。这个过程需要精确的时序控制——tRCD 本质上就是给 SA 的"模拟建立时间"。

3. **Prefetch 是 DDR 各代提速的"作弊码"。** 内部阵列速度有上限（~400 MHz），但可以用更宽的数据通路和串行化来"欺骗"外部接口。每一代 Prefetch 翻倍，接口速率就可以翻倍而不需要内部阵列提速。

4. **Bank 交错是 DRAM 性能的根基。** 不交错 = 总线利用率 < 10%。Bank 越多，调度器越能把内部开销隐藏在并行操作背后。DDR5 增加到 32 Banks 就是为了在高带宽下保持效率。

5. **每一个时序参数背后都有物理原因。** tRCD = SA 放大时间，tRP = BL 均衡时间，tWR = Cs 充电时间，tRFC = 刷新一行的时间，tREFI = 64ms / 行数 / Bank数。理解了这些物理限制，你就不再需要死记硬背参数表——你可以从第一性原理推导出来。
