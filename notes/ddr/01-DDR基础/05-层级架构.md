# 05 层级架构 — 从单颗芯片到完整 DIMM

> 在 01-04 章中，我们讨论的都是单颗 DRAM 芯片内部的事。但一颗 x8 芯片只有 8-bit 宽的 DQ——CPU 需要 64-bit。**怎么用多颗芯片拼出完整的存储系统？**

---

## 9.1 问题的起点：带宽 vs 位宽

单颗 DRAM 芯片的数据位宽是固定的——x4、x8 或 x16。而 CPU 的存储总线宽度通常是 **64-bit**。x8 芯片 → 需要 **8 颗芯片并联**才能匹配 64-bit 总线。这 8 颗芯片组成一个 **Rank**——它们总是同时被选中、执行相同的命令。

但从 1T1C Cell 到 DIMM 插槽——中间经过多个层级。每个层级的存在都有一个明确的物理或经济原因。

---

## 9.2 完整层级链

```
DIMM (内存条, PCB)
├── Rank 0 (一组共用 CA/DQ 总线的芯片, 8颗 x8 = 64-bit)
│   ├── Chip 0 (x8 DDR5)
│   │   ├── Bank Group 0
│   │   │   ├── Bank 0
│   │   │   │   ├── Memory Array × 8 (x8 器件 = 8个 Array 并联)
│   │   │   │   │   ├── Row 0 → [Cell_0_0, Cell_0_1, ..., Cell_0_1023]
│   │   │   │   │   └── Row 1 ~ Row 131071
│   │   │   │   └── Row Decoder + Column Decoder + SA 阵列
│   │   │   ├── Bank 1, 2, 3
│   │   ├── Bank Group 1~7
│   │   └── Control Logic + I/O Buffers
│   ├── Chip 1 ~ Chip 7 (并联)
├── Rank 1 (可选——DIMM 另一面的 8 颗芯片)
└── PMIC (DDR5 专属电源管理芯片)
```

---

## 9.3 Cell → Array → Bank：芯片内部的并行层次

**Cell**：一个 1T1C 单元。1 bit。这是原子单位。

**Memory Array**：Row × Column 个 Cell 组成的矩阵 + Row Decoder + Column Decoder + SA 阵列。一次 ACT 将整行加载到 SA。这是我们 02 章讨论的对象。

**Bank**：**data_width 个 Memory Array 的并联集合。** 对于 x8 器件，一个 Bank = 8 个完全相同的 Array 并联。所有 Array 共享相同的行列地址和命令——它们同步执行相同的操作，各自负责一个 DQ bit。Bank 是**独立操作的基本单位**——不同 Bank 可以处于完全不同的操作阶段。

**Bank Group**：多个 Bank 共享局部 I/O 通路，编入同一个 Group。同一 BG 内的 Bank 切换更慢（**tCCD_L**——共享 IO），跨 BG 的 Bank 切换更快（**tCCD_S**——独立 IO）。这是 DDR4 引入、DDR5 扩展的概念——DDR5 为 8 BG × 4 Bank = 32 Banks。

---

## 9.4 Rank：芯片之间的并行

一个 **Rank** 是一组**共享同一套 CA 和数据总线**的 DRAM 芯片。64-bit 数据总线 + x8 芯片 → **8 颗芯片 = 1 个 Rank**。不同 Rank 通过独立的 **CS_n（Chip Select）**信号来选择——Controller 在同一时刻只与一个 Rank 通信。

Rank 之间的操作可以交错——Rank 0 的数据正在传输时，Rank 1 可以执行 ACT 或 PRE。**但 Rank 间切换需要 ODT 终端切换时间（tRTRS / tOST）**——这会增加一个小的开销。

一个 DIMM 可以有 1 个、2 个或 4 个 Rank。双 Rank DIMM 是最常见的——正反两面各有 8 颗芯片。

---

## 9.5 为什么需要 Bank 交错？

如果只有一个 Bank——每次读写都必须走完 ACT→PRE 的完整 tRC 周期（DDR5-4800 约 46 ns）。在 46 ns 中，数据总线只在 BL16 期间（~3.3 ns）有数据传输。**总线利用率 ≈ 7%**——93% 的时间在空等。

多 Bank 交错解决了这个问题。当 Bank 0 正在做 PRE → ACT 的时——Controller 去访问 Bank 1（它已经处于 ACTIVE 状态，Row 开着）。当 Bank 1 去 PRE 时，Bank 2 刚 ACT 完可以读。**理想情况下，Bus 利用率接近 100%——每个 Bank 的内部 tRC 开销被"隐藏"在其他 Bank 的操作后面。**

DDR5 增加到 32 Banks——就是为了在高带宽下维持这个"隐藏开销"的能力。

---

## 9.6 DIMM 与 Channel

**DIMM**（Dual In-line Memory Module）——我们常说的"内存条"——是把一个或多个 Rank + SPD EEPROM + （DDR5）PMIC 集成在一块 PCB 上的完整模块。

DIMM 通过 **Channel** 连接到 CPU 的 Memory Controller。一个 Channel 包含：64-bit DQ 总线 + DQS 选通信号 + CA 命令地址总线 + CS_n 片选信号。现代 CPU 通常有 2~8 个 Channel——每个 Channel 独立工作，进一步提高总带宽。

SPD（Serial Presence Detect）是 DIMM 上的一颗小 EEPROM——存储了模块的容量、速度、时序参数和制造商信息。Controller 在系统初始化时通过 I2C/I3C 接口读取 SPD——自动配置 DRAM 时序参数（DDR4 使用 I2C，DDR5 升级为 I3C——带内中断，向后兼容 I2C）。

---

## 9.7 一个具体例子

```
系统配置: 2 Channels × 2 DIMMs/Channel × 2 Ranks/DIMM × 8 chips/Rank (x8) × 8 Banks/chip

总 Bank 数 = 2 × 2 × 2 × 8 × 8 = 512 Banks

Controller 可以在任何时刻有最多 512 个独立请求"飞行中"（假设没有 Bank 冲突）
```

这就是现代 DRAM 系统的并行度——512 个独立 Bank，每个都在自己的 tRC 周期中独立运行。

---

## 9.8 本章总结与下章预告

从一颗 1T1C Cell 到 512 Banks 的完整存储系统——每层都在增加并行度：Array 并联 → Bank 交错 → Rank 交错 → Channel 独立。这是一个精心设计的层级结构，每一层的存在都是为了"隐藏"下一层的延迟。

但所有这些操作——ACT、READ、WRITE、PRE、REF——都需要一个统一的"节拍器"来协调。最早期 DRAM 没有时钟——Controller 直接通过 RAS/CAS 信号的电平来触发操作。现代 DRAM 的操作全都锁定在一个时钟信号上——这个从"异步"到"同步"的转变，改变了 DRAM 的一切。下一章，最后一章。
