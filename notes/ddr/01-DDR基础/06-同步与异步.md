# 06 同步与异步 — 时钟如何改变了 DRAM

> 在 02 章中，我们看到所有命令——ACT、READ、WRITE、PRE、REF——都在严格的时序约束下执行。但早期 DRAM **根本没有时钟**。Controller 直接通过控制信号的电平来触发 DRAM 动作。从异步到同步的转变，是 DRAM 历史上最深刻的一次变革。

---

## 10.1 时钟的基本概念

计算机系统中的"时钟"是一个**周期性的方波信号**——规律地在高电平（逻辑 1）和低电平（逻辑 0）之间交替。方波的**上升沿**（从低到高的跳变）和**下降沿**（从高到低的跳变）是所有数字电路操作的"节拍"：

- **系统时钟 / 全局时钟**：位于主板上的主时钟——驱动 CPU、Memory Controller 和大部分外围设备
- **一个时钟周期（tCK）**：从当前上升沿到下一个上升沿的时间。DDR5-4800 下 tCK = 0.416 ns，DDR4-3200 下 tCK = 0.625 ns

## 10.2 异步 DRAM：用控制信号直接"喊话"

在异步 DRAM 中，**没有统一的时钟**。Controller 通过直接发出和取消控制信号（RAS、CAS、WE）来"喊话"给 DRAM——这些信号可以在**任意时刻**到达 DRAM 的引脚。

异步 DRAM 的操作时序由**控制信号本身的时序**决定——而不是由时钟边沿决定。例如：
- RAS 下降沿 → "行地址有效，请打开行"
- CAS 下降沿 → "列地址有效，请读/写数据"
- RAS 和 CAS 必须**同时上升**——不能在保持 RAS 低电平的同时翻转 CAS

这种设计对于低速操作是可行的。但当总线速度提高到 66 MHz 以上时，问题出现了：控制信号到达不同 DRAM 芯片的时间有微小的差异（skew）——异步接口没有"统一的采样沿"来消除这些差异。

《Memory Systems》指出：使 DRAM 成为"异步"的本质是——**Controller 的 RAS 和 CAS 信号直接控制 DRAM 内部的锁存器——这些信号可以在任意时刻到达，时序不确定性完全由 PCB 走线和负载决定。**

---

## 10.3 同步 DRAM：所有操作锁定到时钟边沿

同步 DRAM（SDRAM）的解决方案是将所有操作锁定到一个**统一的时钟沿**上：

- Controller 在 CK 上升沿发出命令 → DRAM 在**下一个**CK 上升沿采样命令
- 数据在**每个**CK 上升沿（SDR）或**双沿**（DDR）被传输
- 所有内部操作的时序——ACT、SA、PRE——都相对于 CK 边沿来计时

同步化的三个好处：
1. **可预测的时序**——所有事件在确定的 CK 周期内发生，消除了"控制信号任意时刻到达"的不确定性
2. **更小的 skew**——所有 DRAM 芯片在相同的时钟沿上采样命令，PCB 走线差异的影响被 CK 周期的"对齐"所吸收
3. **更高的吞吐量**——命令可以更紧密地打包（pipeline），因为下一个命令的时序是确定性的

## 10.4 DLL — 同步 DQS 与 CK 的"桥梁"

同步 DRAM 解决了命令和地址的时序问题——但**数据**的时序仍然是异步的。读操作中，DRAM 返回的数据和 DQS 信号由 DRAM 内部时钟驱动——与 Controller 的全局 CK 之间存在由**芯片内部时钟缓冲**引入的相位延迟。

**DLL（Delay-Locked Loop）**的任务就是消除这个相位延迟。

图 9.37 用上下两张对比图展示了 DLL 的作用：

**没有 DLL（上图）**：DRAM 内部将 CK 缓冲后，用"延迟过的 CK"来生成 DQS 和返回数据。DQS 与 DQ 大致对齐——但 DQS 与全局 CK 之间存在显著的相位移。Controller 无法用全局 CK 来可靠地采样返回数据。

**有 DLL（下图）**：DRAM 仍然内部缓冲 CK——但 DLL 在 DQS 路径上**额外插入了一段延迟**。这个额外延迟的目标是：**CK 缓冲延迟 + DLL 额外延迟 = 恰好一个完整的 CK 周期**。净效果是 DQS 连同 DQ 数据实际上与全局 CK **相位对齐**——"the net effect is to delay the output of the DRAM device by a full clock cycle so that the DQS signal along with the DQ data signals becomes effectively phase aligned to the global clock signal."

> **关键理解**：这解释了为什么 DDR 使用 DLL 后 **CAS Latency 增加了一个周期**——这个额外周期就是 DLL 为了相位对齐而插入的延迟。

## 10.5 从异步到同步的历史转变

《Memory Systems》第 7.2 节用"演进树"展示了从最早的时钟驱动 DRAM 到现代 SDRAM 的完整路径：

```
Clocked DRAM (1960s-1970s)
    ↓
Conventional Asynchronous DRAM (mid-1970s)
    ↓ RAS/CAS 直接控制，每次访问必须 PRE→ACT→COL→PRE
FPM (Fast Page Mode)
    ↓ 允许行在多个 CAS 之间保持打开——"打开一页"
EDO (Extended Data-Out)
    ↓ 输出端加锁存器——数据在总线上保持更久, 可以更早开始 PRE
BEDO (Burst EDO)
    ↓ 内部列计数器——Controller 不需要每列都发 CAS
SDRAM (Synchronous DRAM)
    ↓ 统一时钟驱动所有操作 + 可编程 Burst Length/CAS Latency/多 Bank
DDR SDRAM
    ↓ 双沿传输 + 源同步 DQS + 2n Prefetch
DDR2/DDR3/DDR4/DDR5 ...
```

每一步只增加了**少量逻辑**——FPM 免费（仅消除一个限制），EDO 加了一个锁存器，BEDO 加了一个计数器，SDRAM 加了时钟 + Mode Register。每一步的性能提升约 30%——"Each of these evolutionary changes added only a small amount of logic, yet each improved upon its predecessor by as much as 30% in terms of system performance."

## 10.6 本章总结与系列终章

从 01 章的"一颗电容如何存储 1 个 bit"，到本章的"时钟如何将 DRAM 从异步带入同步时代"——我们走完了一条完整的知识链：

```
1T1C 电容单元 → 阵列组织 → Sense Amplifier → 读写操作 → 刷新 → 时序参数 → Prefetch → 层级架构 → 同步与异步
```

理解 DRAM，本质上就是理解一条物理约束链：**电容小 → 信号弱 → 需要灵敏的 SA → SA 需要时间（tRCD）→ 内部频率有上限 → 需要 Prefetch 来提速 → Prefetch 翻倍加大访问粒度 → 需要多 Bank 交错来维持利用率 → 需要层级架构来组织海量 Bank → 需要一个统一的时钟来协调这一切。**

这条链上的每一个环节，都是下一个环节存在的原因。
